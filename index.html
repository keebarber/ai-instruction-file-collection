Writing a good CLAUDE.md

humanlayer-writing-a-good-CLAUDE-md.md

The best essay on why instruction files fail and how to fix them.

  • Less-is-more: instruction-following degrades as rule count rises — past a point the model ignores all your rules.
  • "Claude is not a linter" — enforce with tools (linters, hooks, tests), reserve the file for what only it can convey.
  • Progressive disclosure: keep the core file tight, load detail on demand.

How to write code — rules cheat-sheet

how-to-write-code-rules.cheatsheet.md

Paste-ready coding rules: a menu to keep only what changes behavior on your codebase.

  • Verify-your-work (highest value): run build/typecheck/lint/test; show evidence, not assertions; fix root cause, not symptom.
  • Avoid: inventing APIs/paths/env vars; deleting tests to pass; new frameworks when one exists; claiming it works without running it.
  • Includes Anthropic's include/exclude table and an adversarial subagent review step.

Writing & voice writing-and-voice/

The anti-slop material: what makes text read as AI-generated, and the positive rules for sounding like a specific person wrote it.

Avoid AI writing — the skill

avoid-ai-writing.SKILL.md

The standout file: a 42-pattern, tiered word/phrase audit you can install as a skill.

  • Three word tiers — always replace (delve, leverage, robust, seamless…), flag if 2+ per paragraph, and flag at high density.
  • Context profiles + a detect-only mode, so it can audit without rewriting.
  • Key insight: the #1 detection signal is structural uniformity, not vocabulary — fix rhythm and specificity.
Verbatim MIT · © Conor Bronsdon source open file

Words & phrases to avoid — quick reference

words-and-phrases-to-avoid.cheatsheet.md

The short list plus the sentence structures to ban — drop straight into a style rule.

  • The hit list: delve, leverage, robust, seamless, tapestry, testament to, game-changer…
  • Banned structures: negative parallelism ("It's not just X — it's Y"), rhetorical-question openers, rule-of-three everywhere, hedge stacking.
  • Formatting tells: heavy em dashes, bold sprinkled throughout, emoji in headers, Title Case Everywhere.
Distilled Adapted (MIT sources) open file

How to write / speak / sound

how-to-write-and-sound.md

The positive counterpart to the avoid-list — a paste-ready voice & tone block.

  • Five principles: vary sentence length, be concrete, have a voice, cut the neutrality, earn your emphasis.
  • Lead with the point; default to plain verbs; one idea per sentence; read it aloud.
  • Warns against over-correction — scrubbing every irregularity pushes text back toward the AI profile.

When an agent should ask a clarifying question versus proceed with a sensible default — and how to ask in a way that doesn't waste the user's time.

When & how an AI should ask questions

when-and-how-to-ask-questions.md

Ask-vs-proceed rules plus the "let Claude interview you" pattern, paste-ready.

  • Proceed when there's an obvious default or the answer is discoverable; ask only when the answer changes what you build.
  • One batched round of concrete multiple-choice questions with a recommended default — not a drip of one-at-a-time.
  • The "interview me" pattern: have the agent dig into edge cases & tradeoffs, then write a complete SPEC.md.

A meta-warning, from the sources themselves

More rules ≠ better behavior. Instruction-following degrades uniformly as instruction count rises — past a point the model starts ignoring all your rules, not just the newest. Keep instruction files short, specific, and observable. Prune ruthlessly, and let deterministic tools (linters, hooks, tests) enforce anything they can.
